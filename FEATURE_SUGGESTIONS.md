## ü§î Questions to Consider

Before implementing any feature:
1. Does it align with core mission (mental wellness)?
2. Is it ethically sound?
3. Does it respect user privacy?
4. Will it create dependency or promote independence?
5. Is it accessible to all users?
6. Does it stand out from competitors?
7. Can it be implemented with current tech stack?
8. What's the effort vs. impact ratio?

## üé® Design Principles for New Features

- **Simplicity First**: Mental wellness users need calm, not complexity
- **Privacy by Default**: All features should be opt-in, data local-first
- **No Dark Patterns**: Never manipulate users for engagement
- **Accessible**: Work for users with disabilities
- **Culturally Sensitive**: Respect diverse backgrounds
- **Evidence-Based**: Grounded in psychology/therapy research
- **Joyful**: Make wellness feel good, not like homework
